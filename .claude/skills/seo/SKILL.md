# SEO Skill

## PageSEO Component

All meta tags are managed via `PageSEO.astro` in the `<head>` slot:

```astro
---
import PageSEO from '@astro-v6/shared/seo/PageSEO.astro';
---

<BaseLayout title="Page Title" description="Description" lang="en">
  <PageSEO
    slot="head"
    title="Page Title - Site Name"
    description="Meta description (max 160 chars)"
    ogImage={new URL('/og/page.png', Astro.site).href}
    ogType="website"
  />
</BaseLayout>
```

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `title` | `string` | required | Page title for OG and Twitter |
| `description` | `string` | required | Meta description |
| `canonicalUrl` | `string?` | `Astro.url.href` | Canonical URL |
| `ogImage` | `string?` | ã | OG image URL (absolute) |
| `ogType` | `string?` | `'website'` | OG type (`website`, `article`) |
| `twitterCard` | `string?` | `'summary_large_image'` | Twitter card type |
| `noIndex` | `boolean?` | `false` | Add noindex/nofollow |

### Generated Output

- `<link rel="canonical">`
- `<meta property="og:title/description/type/url/image">`
- `<meta name="twitter:card/title/description/image">`
- `<script type="application/ld+json">` (WebPage schema)

## Open Graph Images

### Auto-Generation

OG images (1200x630 PNG) are generated at build time using Satori + resvg:

```bash
pnpm --filter starter generate:og   # Starter pages
pnpm --filter blog generate:og      # Blog pages + posts
pnpm build                           # Runs automatically
```

### Configuration

```typescript
// apps/*/scripts/generate-og.ts
const pages = [
  { file: 'home.png', title: 'Astro v6 Starter', description: '...' },
  { file: 'contact.png', title: 'Contact', description: '...' },
  { file: 'de/home.png', title: 'Astro v6 Starter', description: '...' },
  { file: 'de/contact.png', title: 'Kontakt', description: '...' },
];
```

Blog posts are auto-detected from MDX frontmatter and generate `public/og/blog/<slug>.png`.

### Referencing OG Images

```astro
<PageSEO ogImage={new URL('/og/home.png', Astro.site).href} />
<PageSEO ogImage={new URL(`/og/blog/${post.id}.png`, Astro.site).href} />
<PageSEO ogImage={new URL('/og/de/home.png', Astro.site).href} />
```

Always use `new URL(path, Astro.site).href` for absolute URLs.

## Sitemap

Generated by `@astrojs/sitemap` at build time into `sitemap-index.xml` + `sitemap-0.xml`.

### Configuration

```javascript
// Starter: exclude API routes
sitemap({
  filter: (page) => !page.includes('/api/'),
  i18n: { defaultLocale: 'en', locales: { en: 'en', de: 'de' } },
}),

// Blog: exclude RSS
sitemap({
  filter: (page) => !page.includes('/rss.xml'),
  i18n: { defaultLocale: 'en', locales: { en: 'en', de: 'de' } },
}),
```

The i18n config generates `hreflang` attributes for alternate language versions.

### Draft Exclusion

Drafts are excluded from the sitemap because they're filtered in `getStaticPaths()` ã no page is generated, so nothing appears in the sitemap.

## robots.txt

Static files in `apps/*/public/robots.txt`:

```
User-agent: *
Allow: /

Sitemap: https://astrov6.casoon.dev/sitemap-index.xml
```

The Sitemap URL must match the `site` in `astro.config.mjs`.

## Canonical URLs

Auto-resolved from `Astro.url.href` unless explicitly set:

```astro
<PageSEO canonicalUrl="https://astrov6.casoon.dev/specific-page" />
```

## JSON-LD Structured Data

PageSEO includes WebPage schema automatically:

```json
{
  "@context": "https://schema.org",
  "@type": "WebPage",
  "name": "Page Title",
  "description": "Page description",
  "url": "https://astrov6.casoon.dev/page"
}
```

For blog posts, use `ogType="article"` ã the JSON-LD still uses WebPage, but OG type is set to `article`.

## Adding SEO for a New Page

1. Add `<PageSEO>` in the `slot="head"` of `<BaseLayout>`
2. Set `title`, `description`, `ogImage`
3. Add OG image entry in `scripts/generate-og.ts` (both locales)
4. Verify with E2E test: check OG tags, canonical URL, meta description
5. Ensure the page is not filtered out by sitemap config

## SEO Checklist for New Pages

- `<title>` via BaseLayout `title` prop
- `<meta name="description">` via BaseLayout `description` prop
- `<PageSEO>` with title, description, ogImage
- OG image generated (added to `generate-og.ts`)
- Canonical URL resolves correctly
- Page appears in sitemap
- Both locales have matching OG images
